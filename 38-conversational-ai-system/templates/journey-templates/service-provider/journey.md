# Customer Journey Template — Service Provider

This template defines the workflows a complete done-for-you service business
(agency, freelancer, contractor, studio) should have, from inquiry through
project delivery and referral. Service businesses sell time and outcomes, so the
journey is quote-heavy up front and approval/milestone-heavy during delivery.
(Source bullet: add Project Kickoff, Milestone Approvals, Final Deliverable
Handoff, Referral Request workflows.)

## Pre-purchase workflows

### 1. Inbound Inquiry → Quote/Estimate Qualify
**Trigger:** Prospect requests a quote, asks "how much / how long", or fills a
service form
**Purpose:** Scope the job and book the consult — qualify before quoting blind
**Conversation phases:**
- Phase 1 — Acknowledge; capture the job specifics needed to scope
- Phase 2 — Qualify: scope, timeline, budget band, decision-maker
- Phase 3 — Book a scoping call OR send a ballpark + next step
**Success action:** Scoping call booked or estimate request logged; tag
`quote-requested`; brief the provider

### 2. Estimate / Proposal Follow-Up
**Trigger:** Estimate or proposal sent, no decision within 3 days
**Purpose:** Keep the deal warm and surface the real objection
**Conversation phases:**
- Phase 1 — Confirm the estimate was received and clear
- Phase 2 — Surface the blocker (price, timing, scope); offer to phase the work
  rather than cut quality
**Success action:** Re-engaged → route to provider; tag `estimate-followup`

## Purchase moment workflows

### 3. Booking / Deposit Confirmation
**Trigger:** Job accepted / deposit paid / contract signed
**Purpose:** Lock the start and turn "yes" into scheduled work
**Conversation phases:**
- Phase 1 — Confirm deposit + the scheduled start/slot
- Phase 2 — Send the pre-project intake (assets, access, requirements)
**Success action:** Start date confirmed + intake sent; tag `booked`

## Onboarding workflows

### 4. Project Kickoff
**Trigger:** Deposit confirmed / intake submitted
**Purpose:** Align on scope, deliverables, timeline, and communication norms
before work starts
**Conversation phases:**
- Phase 1 — Confirm the kickoff meeting; restate scope + deliverables + dates
- Phase 2 — Set how/when updates and approvals will happen
**Success action:** Kickoff held + scope agreed; tag `kickoff-complete`

## Engagement workflows

### 5. Milestone Approvals
**Trigger:** A milestone/deliverable is ready for client sign-off
**Purpose:** Get fast approvals so the project doesn't stall waiting on feedback
**Conversation phases:**
- Phase 1 — Present the milestone; request structured approval or changes
- Phase 2 — Log change requests; set revision turnaround; collect sign-off
**Success action:** Milestone approved (or revisions scheduled); flag scope-creep
as a change-order opportunity to the provider; tag `milestone-approved`

### 6. Project Status / Check-in
**Trigger:** Mid-project, OR no client activity on a pending approval for 3+ days
**Purpose:** Keep the client confident and unblock waiting items
**Conversation phases:**
- Phase 1 — Proactive status update (what's done, what's next, any blocker)
- Phase 2 — Chase any client-side dependency holding up delivery
**Success action:** Status delivered + blockers cleared; on dissatisfaction,
escalate to provider (sentiment protocol); tag `status-update`

### 7. Final Deliverable Handoff
**Trigger:** Final deliverable complete
**Purpose:** Deliver cleanly, confirm satisfaction, and tee up the next ask
**Conversation phases:**
- Phase 1 — Hand off the deliverable + any docs/access/training
- Phase 2 — Confirm it meets the agreed scope; collect a testimonial
**Success action:** Handoff confirmed + final invoice/testimonial; tag
`delivered`

## Retention workflows

### 8. Maintenance / Retainer Offer
**Trigger:** Project delivered + satisfaction confirmed
**Purpose:** Convert a one-off project into recurring revenue
**Conversation phases:**
- Phase 1 — Propose an ongoing maintenance/retainer or the logical next project
- Phase 2 — Offer loyalty/repeat-client terms (via F28 if a discount, per policy)
**Success action:** Retainer/next-project conversation; tag `retainer-offer`

## Win-back / referral workflows

### 9. Referral Request & Past-Client Reactivation
**Trigger:** Project delivered successfully, OR past client silent 90+ days
**Purpose:** Service businesses grow on referrals — convert delivered outcomes
into introductions and repeat work
**Conversation phases:**
- Phase 1 — Ask for a referral while satisfaction is high; offer a referral perk
  if one exists
- Phase 2 — For dormant clients: a relevant reason to re-engage (new service,
  seasonal need)
**Success action:** Referral captured or reactivation offered; tag
`referral-or-reactivation`

## Notes on Service-Provider-specific tone

- Sell outcomes and reliability; phase scope instead of discounting quality.
- Fast approvals and proactive status updates are the difference between a
  smooth project and an anxious client.
- Flag scope creep as a paid change order, never silently absorb it.
- Never commit timelines or scope the provider must own — escalate those.
- Match the provider's voice from Business Brain (premium-studio vs
  scrappy-responsive).

# PA-21-01: Fog-to-Clarity Framework

**Department:** Clarity Specialist (21)
**SOP ID:** PA-21-01
**Title:** Fog-to-Clarity Framework
**Token:** {{TOKEN}}

---

## 1. Purpose

Turns mental fog ("I don't know what to do," "everything feels stuck") into a clear, actionable picture in one structured conversation. This is the entry-point SOP for the Clarity Specialist department — most clarity sessions begin here when the client cannot name what's actually weighing on them.

## 2. Trigger

- "I don't know where to start" or "I'm spinning my wheels"
- Multiple competing priorities with no clear hierarchy
- Client talks 3+ minutes without landing on a concrete point
- Any session where the presenting issue is confusion, overwhelm, or paralysis

## 3. Inputs / Prerequisites

- Active client session (voice, chat, or async thread)
- Client has expressed some form of stuckness — even if they can't articulate it clearly
- Clarity Log accessible for session artifact storage
- No prior clarity SOP has produced a named action item for this session

---

## 4. DMAIC Procedure

### Define

Ask three questions, capturing verbatim answers only — no paraphrasing, no reframing:

1. **"What's the one thing that feels heaviest right now?"** — Wait for the answer. Don't fill the silence.
2. **"What would 'clear' look like if you had it?"** — This flips the brain from problem-dwelling to solution-imagining.
3. **"What's one thing you DO know, even if it feels small?"** — This is the anchor. Everyone has one. Find it.

If the client deflects any question, return gently: "Take your time. There's no wrong answer here."

### Measure

Map the client's three answers to the Fog-to-Clarity Grid. Score each element 1–5:

| Element | What They Said | Fog Score (1–5) |
|---------|---------------|------------------|
| Heaviest thing | [verbatim quote] | 1 = light haze, 5 = can't see |
| Clear destination | [verbatim quote] | Invert: 1 = crystal clear, 5 = completely hazy |
| Anchor of certainty | [verbatim quote] | 1 = rock solid, 5 = shaky |

**Scoring rule:** A Fog Score of 4+ on the heaviest thing signals emotional charge. Pause and validate before moving to analysis. Say: "That sounds really heavy. Thank you for naming it."

Compute the **Fog Index** = average of the three scores. Record it — it becomes the baseline for measuring clarity progress across sessions.

### Analyze

Classify the fog into one of three buckets:

- **Too Many Doors:** Client has too many options and no criteria to choose. Pattern: "I could do A, or B, or C, or maybe D..."
- **Too Much Noise:** External input is overwhelming. Pattern: "Everyone says I should..." or "I keep reading conflicting advice..."
- **Too High Stakes:** The decision feels permanent and terrifying. Pattern: "If I get this wrong, everything falls apart."

Name the bucket aloud to the client using their words: "It sounds like you're standing in front of too many doors right now — [quote their verbatim confusion]. Let's close a few together."

### Improve

Deploy exactly ONE clarity move matched to the identified bucket. Execute the step — do not describe it:

1. **Too Many Doors → Run the One-Week Rule**
   - Say: "Pick ONE path for just the next 7 days. You're not marrying it — you're dating it."
   - Have the client name the path out loud.
   - Ask: "What would you need to see by day 7 to know this was the right date?"

2. **Too Much Noise → Run the Quiet Hour Prompt**
   - Say: "Of all the voices in your head right now, whose opinion actually matters most?"
   - Have the client name ONE person whose perspective carries weight.
   - Say: "For the next decision on this topic, you only listen to that person — and yourself. Everything else is noise. Can you do that?"

3. **Too High Stakes → Run the Reversible Step**
   - Say: "What's the smallest possible move you could make on this — something you could completely undo by Friday if you wanted to?"
   - Have the client name the step. If it's not reversible, make it smaller.
   - Say: "A reversible step isn't a commitment. It's an experiment. You get to learn without losing."

### Control

Execute the control sequence — verify, log, and report:

1. **Verify — Clarity Statement:** Speak the Clarity Statement aloud. The client confirms or refines:
   > "The heaviest thing is [X], clear would look like [Y], and you already know [Z]. Your next smallest move is [action]. Does that feel right?"
   - Yes → lock as commitment.
   - No → return to Define step 2 and re-anchor on "what would clear look like."

2. **Log:** Write the session artifact to the Clarity Log immediately:
   ```
   [Date] — Fog-to-Clarity Session
   Heaviest: [quote] | Clear destination: [quote] | Anchor: [quote]
   Fog Index: [score]/5 | Bucket: [Doors / Noise / Stakes]
   Next move: [action]
   ```

3. **Report:** If the Fog Index was 4+ and is now ≤2, flag as "breakthrough session" to the Operator. If Fog Index remains 4+ after three sessions, escalate per the escalation table below.

## 6. Metrics

| Metric | Target |
|--------|--------|

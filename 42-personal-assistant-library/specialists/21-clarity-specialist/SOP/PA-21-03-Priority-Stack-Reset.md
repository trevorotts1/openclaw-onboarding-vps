# PA-21-03: Priority Stack Reset

**Department:** Clarity Specialist (21)
**SOP ID:** PA-21-03
**Title:** Priority Stack Reset
**Token:** {{TOKEN}}

---

## 1. Purpose

When everything is urgent, nothing is. Clients drowning in to-do lists, competing demands, and "I should be doing X but Y is on fire" need a full stack reset — not another prioritization framework to learn, but a quick, brutal, compassionate re-sort of what actually matters right now. This SOP follows PA-21-01 when the fog has cleared enough to see the full plate but the plate is overflowing.

## 2. Trigger

- Client lists 8+ things they "need to do"
- Client says "I'm behind on everything" or "I don't even know where to start"
- Client is working on low-impact tasks while high-impact ones sit untouched
- Client's calendar and stated priorities visibly don't match
- Client hasn't made meaningful progress on their #1 goal in 2+ weeks

## 3. Inputs / Prerequisites

- Active client session
- Client can articulate what's on their plate (if they can't name their tasks, deploy PA-21-01 first)
- Calendar access (client's or shared view) for time-blocking in the Improve phase
- Clarity Log accessible for stack artifact

---

## 4. DMAIC Procedure

### Define

1. **Full Brain-Dump:** Ask the client to list EVERYTHING on their plate — professional, personal, obligations, aspirations, dreads. No filtering, no organizing yet. Say: "Just get it all out. We'll sort it together."
2. **The Three-Item Filter:** Once the list is complete, ask:
   > "If you could only accomplish THREE things in the next two weeks and still feel proud of yourself, what would they be?"
3. Those three become the Priority Stack. Everything else moves to the Parking Lot — not deleted, just deferred. Say: "These three are your yes. Everything else is 'not right now.'"

### Measure

Score each of the three Priority Stack items on two dimensions (1–5), with the client providing every score:

| Priority | Impact (1–5) | Urgency (1–5) | Stack Rank |
|----------|-------------|---------------|------------|
| [Item 1] |             |               |            |
| [Item 2] |             |               |            |
| [Item 3] |             |               |            |

**Impact** = "How much does this move the needle on what you ACTUALLY care about?" (Not what looks good to others. Their real values.)

**Urgency** = "What happens if this waits two more weeks?" (5 = genuine disaster, 1 = nothing meaningful changes.)

**Stack Rank = Impact × Urgency.** Higher number = higher priority. If two items tie, Impact breaks the tie — Impact always wins over urgency. Urgent-but-low-impact is the productivity trap. Name it if you see it: "This feels urgent, but does it actually move YOUR needle?"

### Analyze

Now confront the Parking Lot with the client. This is where the real reset happens:

1. **The "Should" Audit:** Say: "Look at everything in your Parking Lot. Which of these are you holding onto because you feel like you SHOULD — not because they actually matter to YOU?"
2. Go item by item. For each one the client identifies as a "should," ask: "Whose expectation is this meeting?"
3. **Celebrate the deletions.** Every "no" makes room for the three "yeses." Say: "That's [count] things you just freed yourself from. How does that feel?"

If genuinely everything in the Parking Lot is essential, the client may need delegation — not prioritization. Flag to Operator: "Client at capacity; delegation opportunity."

### Improve

Build the Two-Week Sprint with concrete, calendar-backed actions:

1. **Priority 1 — Time-Block:**
   - Open the client's calendar (or have them open it).
   - Identify and book 3 specific time slots THIS WEEK. Name each slot: "Tuesday 9-11 AM: [P1 task name]."
   - Confirm the client sees the blocks on their calendar. "Do you see it? That time is now spoken for."

2. **Priority 2 — Momentum Task:**
   - Ask: "What's the ONE sub-task on this that would create momentum — the thing that, once done, makes the rest feel easier?"
   - Have the client name it specifically.
   - Schedule it on a specific day and time. "Wednesday 2-3 PM: [momentum task]."

3. **Priority 3 — Minimum Viable Completion:**
   - Ask: "What's 'done enough' for now? Not perfect — done enough to call it complete for this sprint."
   - Define the finish line in observable terms. "I'll know it's done when [observable signal]."
   - Schedule one block for it.

The sprint is specific: what, when, for how long. Vague intentions ("I'll try to work on it") do not count and must be rejected. If the client says "I'll try," respond: "Let's get more specific. What does 'trying' look like on Thursday at 10 AM?"

### Control

Execute the three-part control sequence:

1. **Verify — Reset Statement:** Speak the Reset Statement. The client acknowledges:
   > "For the next two weeks, your stack is: [P1], [P2], [P3]. Everything else is parked. If something truly catches fire, we'll re-sort — but you don't have to borrow tomorrow's urgency today. Does that hold?"

2. **Log:** Write the Stack Artifact immediately:
   ```
   [Date] — Priority Stack Reset
   Two-Week Stack:
     1. [Priority 1] — [3 calendar slots with dates/times]
     2. [Priority 2] — [momentum task + scheduled date/time]
     3. [Priority 3] — [minimum viable completion + scheduled date/time]
   Parking Lot: [count] items deferred | [count] "should" items eliminated
   Next check-in: [date + 14 days, exact time]
   ```

3. **Report:** Schedule the 14-day check-in with the client. At that check-in: celebrate every completion, re-assess the Parking Lot, and build the next sprint. If Priority 1 was not touched, escalate per the table below.

## 5. CTQ Checks

☐ Full brain-dump completed — everything on the plate captured without filtering
☐ Top 3 priorities selected by {{TOKEN}} with conviction (not hesitation)
☐ Every Priority Stack item scored on Impact and Urgency by the client
☐ Calendar slots blocked for all 3 priorities (≥3 for P1, ≥1 each for P2 and P3)
☐ Parking Lot reviewed and "should" items explicitly eliminated with client confirmation

---

## 6. Metrics

| Metric | Target |
|--------|--------|
| Priority completion rate at 14-day check-in (% of stack items completed) | ≥67% (2 of 3) |
| "Should" items eliminated per session | ≥3 |
| Calendar block adherence (client reports working during booked blocks) | ≥75% |

## 7. Escalation

| Signal | Route To |
|---|---|
| {{TOKEN}} cannot narrow to 3 priorities after two attempts | Deploy PA-21-01 (Fog-to-Clarity) first |
| Every Parking Lot item genuinely essential | Operator — delegation opportunity |
| Priority 1 stalls across 2 consecutive sprints | Specialist 27 (Focus & Completion) |
| {{TOKEN}} reports overwhelm despite stack reset | Specialist 08 (My Coach) |

---

## 8. Definition of Done

{{TOKEN}} has 3 ranked priorities with calendar-blocked actions, a cleared Parking Lot with eliminated "should" items, and a 14-day check-in scheduled at a specific date and time.

---

## 9. Tone & Persona Note

Brutal-but-compassionate editor. "Every yes to the wrong thing is a no to what matters" — say it once, then act on it. Celebrate deletions as loudly as completions — freeing yourself from a "should" is a win. Faith-respectful; culturally fluent for Black women professionals.

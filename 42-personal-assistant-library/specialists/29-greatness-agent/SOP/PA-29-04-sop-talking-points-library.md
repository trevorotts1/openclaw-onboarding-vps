# PA-29-04: Talking Points Library

**Title:** Talking Points Library
**SOP ID:** PA-29-04
**Purpose:** Build and maintain a searchable, ready-to-deploy library of key messages, stories, and proof points so {{OWNER_NAME}} walks into every media interview, podcast, panel, investor meeting, or networking event with crisp, confident, memorable answers — never searching for words under pressure.
**Trigger:** Media or speaking opportunity confirmed; new accomplishment or venture milestone achieved; quarterly refresh; {{OWNER_NAME}} says "what do I say about...?"
**Inputs / Prerequisites:** Intake document (PA-29-01), Bio Suite (PA-29-03), Living Resume (PA-29-02), specific opportunity brief if applicable, {{TOKEN}} for persona voice.

### DMAIC Procedure

- **Define:** Identify the specific context requiring talking points. Is it a podcast interview (long-form, conversational, story-driven)? A panel (short, punchy, 90-second answers)? An investor pitch (credibility + vision, numbers-heavy)? A networking event (memorable 30-second introduction)? Media interview (on-record, quotable, bridge-to-key-message)? The format determines the structure. Confirm: what's the ONE thing {{OWNER_NAME}} wants the audience to remember? Restate: "We're building the words so you can focus on the connection — not on remembering what to say."

- **Measure:** Gather the raw material. Pull 5-7 signature accomplishment stories from the intake document. Extract 3-5 key statistics or proof points from the Living Resume. Identify 2-3 current initiatives or ventures to highlight. Map the likely questions: what will the interviewer, investor, or audience most want to know? For each question category, pull the strongest supporting evidence. A talking point without proof is an opinion; a talking point with a number is a fact.

- **Analyze:** Audit {{OWNER_NAME}}'s past speaking or interview performance if available. What patterns emerge? Does {{OWNER_NAME}} deflect credit, use qualifiers ("just," "only," "kind of"), speak in generalities instead of specifics? These are the under-claiming tells the talking points must pre-empt. For each likely question, assess whether {{OWNER_NAME}}'s instinctive answer matches the strength of the actual accomplishment — if the instinct is weaker, the talking point must bridge the gap.

- **Improve:**
  1. Build the core message framework: (a) Origin Story — why {{OWNER_NAME}} does this work, in 60 seconds. (b) Credibility Stack — 3 proof points that establish why {{OWNER_NAME}} is the person to listen to. (c) Current Work — what {{OWNER_NAME}} is building or driving right now, with a specific result. (d) Vision — where this is going, and why it matters. (e) Call to Action — what {{OWNER_NAME}} wants the audience to do next.
  2. Write question-specific talking points using the ANSWER-BRIDGE-PROOF formula: directly answer the question, bridge to the key message, anchor with a specific proof point or story. Example: "Was it hard starting your first company?" → "It was — but the real challenge wasn't the startup, it was [specific insight]. When we [specific action], we saw [result]. That's what shaped how I build things now."
  3. Prepare 3 "go-to stories" — narrative anecdotes from the intake document that can flex across multiple questions. Each story has: setup (context), action (what {{OWNER_NAME}} did), result (quantified outcome), insight (what was learned).
  4. Build the "pivot library" — bridging phrases that move from any question to {{OWNER_NAME}}'s key messages: "What that taught me was..." "The more important question is..." "Here's what I'd want your audience to know..."
  5. Compile into a searchable document organized by context (interview, pitch, panel, networking) and question category. Deliver to {{OWNER_NAME}} with a practice recommendation: "Read these aloud once before the event. The goal isn't memorization — it's familiarity. You'll sound like yourself, just sharper."

- **Control:** Test every talking point for the "credibility gut check": if a skeptical listener heard this, would they believe it? Talking points must be ambitious enough to impress and specific enough to be unassailable. Run the under-claim scan on the full library (PA-29-05). After each live use, debrief with {{OWNER_NAME}}: which points landed? Which felt unnatural? Update accordingly. The library is a living asset — it improves with every real-world test.

### CTQ Quality Checks
- [ ] Core message framework fully populated (Origin, Credibility, Current Work, Vision, CTA)
- [ ] Minimum 3 go-to stories with setup-action-result-insight structure
- [ ] Every talking point anchored to a specific proof point from the intake document
- [ ] Pivot library contains at least 5 bridging phrases

### Metrics
1. **Library coverage:** Percentage of likely question categories with prepared talking points. Target: ≥90%.
2. **Proof point density:** Percentage of talking points containing a specific number, story, or verifiable claim. Target: 100%.
3. **Post-event confidence score:** {{OWNER_NAME}}'s self-reported comfort level after using the library. Target: ≥8/10.

### Escalation
If {{OWNER_NAME}} consistently replaces prepared talking points with weaker, self-diminishing answers in live settings, flag for under-claim intervention (PA-29-05). This is a pattern, not a preparation failure. If a high-stakes opportunity (national media, major conference, key investor) requires talking points beyond the scope of the existing library, escalate for expanded research and persona support.

### Definition of Done
Complete talking points library organized by context and question category, delivered to {{OWNER_NAME}} before the event, tested in at least one live setting, and updated based on feedback.

### Tone & Persona Note
Speechwriter voice: sharp, strategic, grounded in evidence. You are the person who makes {{OWNER_NAME}} sound like the best version of themselves — not a scripted version, but the version that remembers every accomplishment and says it out loud. "You already know this stuff. The talking points just make sure you say it — instead of deflecting, qualifying, or skipping the good part. The good part IS the point."

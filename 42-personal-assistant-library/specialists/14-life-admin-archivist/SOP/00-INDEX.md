## Reading Order

New to the department? Read SOPs in this order:

1. **PA-14-01 (Document Filing System)** — The foundation. How everything is named, organized, and stored. Every other SOP depends on a healthy filing system.
2. **PA-14-03 (Account & Credential Hygiene)** — The security backbone. Understanding pointer hygiene (never plaintext secrets) is non-negotiable before handling any account information.
3. **PA-14-04 (Important-Info Capture)** — The input mechanism. How information enters the system. You will use this constantly.
4. **PA-14-02 (Information Retrieval)** — The output mechanism. How to find things fast. Builds on the filing system from PA-14-01.
5. **PA-14-05 (Renewal & Expiration Tracking)** — The prevention layer. Keeps the system healthy over time by catching deadlines before they become emergencies.

## DMAIC Compliance

All SOPs follow the DMAIC structure: Define, Measure, Analyze, Improve, Control. Each includes CTQ binary checks, metric targets, escalation rules, and a Definition of Done.

## Critical Rule Summary

| Rule | SOP | Why |
|---|---|---|
| Date leads every filename (YYYY-MM-DD) | PA-14-01 | Sort-by-name = sort-by-time. Universal compatibility. |
| Zero plaintext secrets — pointers only | PA-14-03 §2 | One leaked secret is a disaster. The inventory is the map, not the vault. |
| Capture within 90 seconds of hearing it | PA-14-04 §2 | Information fades. Capture reflex is non-negotiable. |
| Every renewal gets a 30-day AND 7-day warning | PA-14-05 §1 | Lead time is the difference between quiet task and screaming emergency. |
| If not found in 2 minutes, flag it — don't hunt indefinitely | PA-14-02 §3 | Retrieval time is valuable. Unfound documents need system fixes, not more search. |

## Purpose

To capture, structure, and store any piece of important information the moment it surfaces — so nothing falls through the cracks between "I should write that down" and "what was that thing again?"

## Scope

Applies to verbal instructions, meeting takeaways, decisions, numbers, dates, names, process changes, and any unstructured information that {{TOKEN}} or a team member mentions needs to be remembered or acted upon.

## Procedure

### 1. Capture Triggers — When to Activate

You hear any of these phrases? **Capture immediately.** Don't wait for a "good time."

| Trigger Phrase | Action |
|---|---|
| "Remind me to…" | Create a dated reminder + file the info |
| "The password for… is…" | Route to secrets store; record pointer (PA-14-03) |
| "Starting next month, we're switching to…" | Calendar event + process note |
| "My new [account/insurance/policy] number is…" | File in correct folder (PA-14-01) |
| "Make sure we don't forget…" | Flag as high-priority; confirm out loud |
| "Here's what we decided…" | Decision log entry with date + context |
| "The deadline for… is…" | Calendar + renewal tracker (PA-14-05) |

### 2. The 90-Second Capture Flow

When triggered, you have 90 seconds to complete this flow:

1. **Capture raw** (0–15 sec): Write it down EXACTLY as stated. No editing, no formatting, no "I'll clean it up later." Voice memo, quick note, or scratch file.
2. **Classify** (15–30 sec): What kind of information is this? Decision? Date? Credential? Process? Contact? Map it to its home folder or tool.
3. **File** (30–60 sec): Put it where it belongs using the filing system (PA-14-01). If it's a credential, use pointer-only (PA-14-03). If it's a date, add to the renewal tracker (PA-14-05).
4. **Confirm** (60–90 sec): Echo back to {{TOKEN}}: "Got it — [summary] filed under [location]. You'll get a reminder on [date]." This closes the loop and surfaces any misunderstanding immediately.

### 3. Decision Log

Every significant decision gets a one-line entry in the Decision Log:

```
YYYY-MM-DD | Context | Decision | Decided By | Rationale (1 sentence)
```

The Decision Log lives at `Business/Decision-Log.md`. It answers "why did we choose X?" — a question that comes up more often than anyone expects.

### 4. Information Types & Routing

| Information Type | Route To |
|---|---|
| Date / Deadline | Calendar + PA-14-05 tracker |
| Credential / Access | Secrets store → pointer in PA-14-03 ledger |
| Decision | Decision Log |
| Process change | Update the relevant SOP |
| Contact info | Apple Contacts + CRM if business |
| Reference number | File under appropriate category (PA-14-01) |
| "Nice to know" (not actionable) | Personal knowledge base / Notion |

### 5. Capture Gaps — When You Missed Something

If you realize something wasn't captured:

1. **Reconstruct immediately** while memory is fresh. Even partial information is better than nothing.
2. **Flag the gap.** Ask {{TOKEN}}: "I missed capturing [topic] earlier. Can you restate it?"
3. **Identify the failure.** Why was it missed? No trigger phrase? Distraction? Wrong tool open? Fix the system, not yourself.

### 6. Weekly Capture Review

Part of the Sunday filing ritual (PA-14-01 §5): review all captures from the week. Check that:
- Every capture was filed correctly
- Decision Log entries are complete
- No dangling "remind me to…" items without reminders actually set

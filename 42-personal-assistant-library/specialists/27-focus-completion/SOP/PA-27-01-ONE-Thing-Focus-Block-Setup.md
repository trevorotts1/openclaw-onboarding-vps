## Objective

Establish a protected, single-outcome focus block where exactly ONE priority task advances to completion  --  no switching, no checking, no side-quests.

## Why This Exists

Rubinstein, Meyer, and Evans (2001) demonstrated that task-switching incurs a measurable cognitive "switch cost"  --  the brain does not parallel-process; it serial-switches, and every switch burns recovery time. For complex tasks, switch cost compounds: interrupt twice in an hour and you may lose 40% of productive throughput. The ONE-Thing block eliminates switch cost entirely for a defined window.

## DMAIC Procedure

### Define
Identify the single highest-leverage task for the session. If everything is priority, nothing is. Pick ONE. Format the outcome as: "By [time], I will have [completed deliverable]." The deliverable must be binary-testable  --  you or an observer could confirm instantly whether it is done.

### Measure
Baseline your current output: how many distinct tasks did you touch in your last 90-minute work window? Count them. Record your current switch count before the first block  --  you need a starting point to prove improvement.

### Analyze
Each task-switch costs 15–25 minutes of deep-focus recovery (Rubinstein et al., 2001). Multiply your switch count by 20 minutes. That is your attention tax  --  the invisible cost of fragmented attention, now made visible. If you touched 5 tasks in 90 minutes, you lost approximately 100 minutes of productive depth to switching overhead alone.

### Improve
1. **Impact Filter Selection:** Before every block, ask aloud: "If I complete ONLY this today, will the day feel productive?" If the answer is no, you have the wrong ONE thing. Pick the task whose completion creates the most downstream relief or momentum  --  that is the task your brain is already carrying as weight.
2. **Environment Pre-Flight Checklist:** Execute the 5-step Pre-Block Setup (see Procedure section below) in exact sequence without skipping any item. Skipping phone removal or notification silencing is the #1 predictor of block failure. Treat this like a pilot's pre-flight checklist: every item, every time, no exceptions.
3. **Scratch Pad Redirect Protocol:** When a non-task thought arises ("don't forget to email X"), write it on the scratch pad within 10 seconds and return visual focus to the ONE task. The thought is captured; it will not be lost. What sabotages focus is the fear of forgetting  --  not the thought itself. The scratch pad eliminates that fear.
4. **Halfway Reset Ritual:** At the 45-minute timer alarm, stand for 60 seconds, re-read the ONE outcome statement aloud, take one deep intentional breath, and sit back down. Do not check any device during this break. This micro-reset prevents the second-half drift that causes most partial completions.
5. **Condition Replication Loop:** After completing 4 blocks in a week, review: which block had the highest subjective depth score? Replicate those exact conditions  --  time of day, location, setup ritual, task type  --  for next week's highest-leverage blocks. Your brain already showed you when and how it works best. Listen to it.

### Control
- **Verify:** At block end, confirm outcome status  --  COMPLETE, PARTIAL (with % estimated), or ABANDONED (with reason). Log the result immediately in PA-27-02. Do not trust memory; log it before checking any communication channel.
- **Log:** Record in PA-27-02: block duration, switch count (target: 0), and subjective depth score (1–10). Also note which Improve step was most impactful for this specific block.
- **Report:** Weekly: calculate clean block completion rate against the 80% target. Flag any block below 5/10 depth score for pattern review. Flag 3+ consecutive ABANDONED blocks for immediate Specialist escalation.

## Procedure

### Pre-Block Setup (5 minutes)
1. Write the ONE outcome on a physical sticky note or a plain-text editor. Format: "By [time], I will have [completed deliverable]."
2. Close every application and browser tab not required for that single deliverable.
3. Silence phone, smartwatch, desktop notifications, Slack, Telegram, email  --  all of them. Not "do not disturb"  --  OFF.
4. Set a visible timer for 90 minutes. No pausing. No checking time.
5. Place water and any needed physical items within arm's reach so you never leave the chair.

### During the Block (90 minutes)
1. Work ONLY on the defined outcome. A thought about another task? Write it on a scratch pad and return immediately  --  do NOT open the other task.
2. If someone interrupts in person: "I'm in a focus block until [time]. I'll find you then." Do not negotiate.
3. If you catch yourself drifting: acknowledge it without judgment, return to the ONE thing. This IS the practice.
4. At the halfway alarm (45 min), stand for 60 seconds. Do not check any device. Sit back down.

### Post-Block (5 minutes)
1. Mark the outcome: COMPLETE, PARTIAL (note %), or ABANDONED (note why).
2. Log the block result in PA-27-02 (Task Completion Tracking).
3. Take a genuine 15-minute break before the next block. No "productive" break  --  walk, stretch, stare out a window.

## Edge Cases

- **Emergency interrupt:** If it's genuinely urgent (someone bleeding, server down, client crisis), kill the block immediately, handle it, then restart a fresh block. Do not "resume"  --  the cognitive state is gone.
- **Stuck / blocked on the task:** Spend 5 minutes defining the exact blocker. If unblocking requires someone else, send the request and switch to the next-highest-leverage ONE thing. Log the block as PARTIAL with a blocker note.
- **Task finished early:** Do NOT start a new one. Use remaining time to review, refine, or document the completed work. The block is sacred space, not a productivity race.

## CTQ (Critical to Quality)  --  Binary Checks
- [ ] ONE outcome written in "By [time], I will have [deliverable]" format before block starts
- [ ] All notifications OFF (not "do not disturb") for full 90-minute block
- [ ] Zero task switches during the block (scratch pad use only for capture, not execution)
- [ ] Post-block outcome marked as COMPLETE, PARTIAL, or ABANDONED with note

## Metrics
| Metric | Target |
|---|---|
| Clean block completion rate | ≥ 80% |
| Switch count per block | 0 |
| Subjective depth score (1–10) | ≥ 7 |

## Escalation
If clean block completion rate drops below 60% for 2+ consecutive weeks, escalate to Specialist 09 (Emotional Support & Wellbeing) for overwhelm assessment. If switching persists despite full protocol installation, escalate to Specialist 24 (Superwoman Syndrome) for overfunctioning-pattern evaluation. Notify Master Orchestrator if 3+ consecutive blocks are ABANDONED without clear blocker resolution.

## Definition of Done
A ONE-Thing Focus Block is complete when the defined outcome is marked COMPLETE and the block result is logged in PA-27-02 with duration, switch count, and depth score.

Celebrate completed blocks warmly; investigate incomplete blocks with curiosity, never judgment. "You protected 90 minutes for your most important work. That's the practice  --  and you showed up for it."

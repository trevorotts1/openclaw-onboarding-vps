## 3. Knowledge Verification Questions

Answer each question from memory (no looking up). Correct answers are provided for self-check.

**Q1: What are the two output files the script produces?**
> Correct: `<output>.json` and `<output>.md`

**Q2: What is the clip duration for `veo-3-1`?**
> Correct: 8 seconds (fixed)

**Q3: What three duration options does Sora support?**
> Correct: 10s, 15s, 25s

**Q4: Which file is the canonical source of truth for model durations and pricing?**
> Correct: `scripts/model-database.json`

**Q5: What should the agent do before calling the script — and in what order should it ask the user?**
> Correct: Ask the user ONE AT A TIME: (1) topic, (2) duration, (3) platform, (4) budget, (5) preferred model.

**Q6: What is the gateway restart rule during installation?**
> Correct: Never trigger it autonomously. Stop, notify the user, instruct them to type `/restart` in Telegram, and wait for confirmation.

**Q7: What platform/model pairing is recommended for TikTok/Reels/Shorts?**
> Correct: `veo-3-1` (8s clips, vertical, under 60s)

**Q8: What platform/model pairing is recommended for YouTube (horizontal, 60s–10min)?**
> Correct: `sora-25s` (fewer cuts)

**Q9: Which two core files is this skill allowed to update?**
> Correct: `TOOLS.md` and `MEMORY.md` only.

**Q10: What does the agent do with the template placeholder prompts generated by the script?**
> Correct: Replace them with specific, visual, production-ready descriptions. Do NOT use placeholders as final output.

**PASS** if 9–10 correct. **FAIL** if fewer than 8 correct — re-read `SKILL.md` and `INSTRUCTIONS.md`.

## 4. Live Behavior Test

Run these commands from the skill folder. All must complete without errors.

### 4a. List available model IDs

```bash
python3 -c "from scripts.model_database import list_models; print('\n'.join(list_models()))"
```

**PASS** if output includes at minimum:
- `veo-3-1`
- `veo-8s`
- `sora-10s`
- `sora-15s`
- `sora-25s`
- `kling-3`
- `kling-5s`
- `kling-10s`
- `kling-15s`
- `kling-20s`
- `seed-dance`
- `wan-2-6`

**FAIL** if command errors out or list is empty.

### 4b. Cost calculation

```bash
python3 -c "
from scripts.model_database import calculate_cost
result = calculate_cost('sora-25s', 300)
print(result)
assert result['num_clips'] == 12, f'Expected 12, got {result[\"num_clips\"]}'
assert result['clip_duration'] == 25, f'Expected 25, got {result[\"clip_duration\"]}'
assert result['total_cost'] == 48.0, f'Expected 48.0, got {result[\"total_cost\"]}'
print('cost check PASSED')
"
```

**PASS** if output ends with `cost check PASSED`.
Expected: 300s ÷ 25s = 12 clips × $4.00 = $48.00

### 4c. Generate a test storyboard

```bash
python3 scripts/create_storyboard.py \
  --duration 40 \
  --model veo-3-1 \
  --topic "QC Test" \
  --output qc_test
```

**PASS** if:
- No errors printed
- `qc_test.json` created
- `qc_test.md` created
- Output includes `Segments: 5` (40s ÷ 8s = 5 clips)
- Output includes `Estimated cost: $2.0`

### 4d. Validate JSON output structure

```bash
python3 -c "
import json
with open('qc_test.json') as f:
    sb = json.load(f)
assert 'project' in sb
assert 'model' in sb
assert 'calculations' in sb
assert 'segments' in sb
assert len(sb['segments']) == 5
assert sb['model']['id'] == 'veo-3-1'
assert sb['calculations']['num_segments'] == 5
print('JSON structure PASSED')
"
```

**PASS** if output ends with `JSON structure PASSED`.

### 4e. Validate Markdown output

```bash
grep "Segment 1" qc_test.md && \
grep "Segment 5" qc_test.md && \
grep "veo" qc_test.md && \
echo "markdown check PASSED"
```

**PASS** if output ends with `markdown check PASSED`.

### 4f. Unknown model error handling

```bash
python3 scripts/create_storyboard.py \
  --duration 30 \
  --model fake-model-xyz \
  --topic "Test" \
  --output should_not_exist
```

**PASS** if:
- Output includes `Unknown model 'fake-model-xyz'`
- Output includes a list of available models
- `should_not_exist.json` is NOT created

### 4g. Cleanup test artifacts

```bash
rm -f qc_test.json qc_test.md
```

## 5. Anti-Pattern Checks

Verify the agent did NOT do any of the following during or after installation.

| Anti-Pattern | Check | PASS if |
|---|---|---|
| Modified unauthorized core files | `git -C ~/.openclaw diff --name-only HEAD` | Only `TOOLS.md` and `MEMORY.md` changed |
| Triggered gateway restart autonomously | Review install log/transcript | No `openclaw gateway restart` was run without user confirmation |

| Asked all onboarding questions at once | Review conversation transcript | Questions were asked one at a time |
| Used `topaz` as a generation model | Attempt `--model topaz-*` and confirm it is absent | `topaz` has no durations; `list_models()` does not include it |
| Skipped reading all five .md files | Review install transcript | SKILL.md → CORE_UPDATES.md → INSTALL.md → INSTRUCTIONS.md → EXAMPLES.md were all read |

**FAIL** if any anti-pattern is detected.
